Цей проєкт демонструє використання Liquibase та Docker для автоматизації роботи з базою даних вебплатформи для обміну навичками між користувачами на основі інтелектуального підбору партнерів.
Основні компоненти
Для роботи проєкту використовується база даних PostgreSQL 17, яка розгортається в Docker-контейнері. Керування структурою таблиць створюється через Liquibase, що дозволяє відстежувати зміни та виконувати відкат (rollback) до попередніх версій схеми.
Як працювати з проєктом
Для запуску середовища необхідно виконати команду docker-compose up -d. Після того як контейнер стане активним, можна перевірити стан міграцій командою liquibase status. Для створення таблиць та застосування нових змін використовуйте команду liquibase update. Якщо необхідно скасувати останню дію, скористайтеся командою liquibase rollbackCount 1.
Опис структури
У папці db/changelog міститься головний файл конфігурації міграції, описано створення таблиці навичок (skills) та подальше додавання нових колонок для профілів користувачів. Файл docker-compose.yml містить налаштування порту, ім'я бази даних skill_exchange_db та паролі, а liquibase.properties забезпечує зв’язок інструменту міграції із сервером PostgreSQL.
